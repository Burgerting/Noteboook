# 對話歷史紀錄存檔 (Chat History)

## 2026-07-22 對話紀錄

### 需求與問題 1：新增信用卡帳單管理與 Gmail 自動掃描
* **說明**：使用者希望能自動將每個月的信用卡帳單載入至記帳本中，減少手動輸入的時間。
* **調整**：
  - 新增 `CreditCardTab.tsx` 頁籤元件，支援按月呈現與新增信用卡消費。
  - 新增 `gmailApi.ts`，利用 Google API 搜尋過去 45 天內含有「電子帳單」、「信用卡」、「自動扣繳」的信件。
  - 整合 `pdfjs-dist`，透過使用者輸入的身分證字號解鎖銀行加密的 PDF 帳單檔，並自動解析「應繳總額」。
  - 在前端實作 `idStorage.ts` 以安全儲存身分證字號於本機，避免上傳至任何後端。
  - 更新 `accountingSync.ts` 將信用卡資料寫入並同步至 Google Drive 雲端個人空間。

## 2026-07-23 對話紀錄

### 需求與問題 1：Google Drive 共用資料夾選擇器優化
* **說明**：使用者不希望每次都用貼上網址或資料夾 ID 的方式加入共用記帳本。
* **調整**：實作了 Google Picker API (`useDrivePicker.ts`)，讓使用者能透過彈出視窗直覺地點選雲端硬碟中的資料夾，並自動帶入資料夾名稱。

### 需求與問題 2：電腦與手機的離線/本機端儲存 (雙軌儲存機制)
* **說明**：使用者希望輸入的每一筆帳務 (包含共用) 都能在沒有網路時依然能看得到，且電腦端也要有實體檔案備份。
* **調整**：
  - 開發 `localFsPlugin` (Vite Plugin) 讓本機開發時能直接把 JSON 寫入實體的 `D:\Eating\小工具\記帳記事本\local_drive`。
  - 實作雙軌儲存機制：如果偵測到處於手機端或公開網頁版 (實體檔案寫入失敗時)，自動無縫退回使用瀏覽器的 `localStorage` 作為離線快取，確保任何情境下都能離線瀏覽資料。
  - 加入信用卡「年度總額」加總顯示功能與千分位逗號格式化。

### 需求與問題 3：APP 打包與 iOS 限制處理
* **說明**：使用者希望能產出 Android (APK) 與 iOS 的安裝檔。
* **調整**：
  - 協助修改 `build_apk.ps1`，成功編譯出 Android 的安裝檔 (`SyncApp_android.apk`)。
  - 說明了 Apple 必須使用 Mac (Xcode) 編譯 `.ipa` 的原廠硬體限制。

### 需求與問題 4：免伺服器公開發布 (GitHub Pages + PWA)
* **說明**：使用者希望即使手機出門在外、不連家中 WiFi 也能隨時記帳，且要有 APP 的體驗。
* **調整**：
  - 將專案連線路徑與系統設定修改為相容 GitHub Pages (`Noteboook`)。
  - 加入 SPA 的 404 路由修復邏輯。
  - 協助使用者將記帳本正式部署到 GitHub Pages 上，讓使用者可以直接透過手機瀏覽器將網頁「加入主畫面 (PWA)」，獲得無縫的類原生 APP 體驗。
